Multi-Granularity Joint Computing Framework

A modern, structured implementation for Citizen Science Project Management

1. Website Purpose & Users

Primary Users

  • Development team (5 MIS students)
  • Instructor (for evaluation)
  • Client (scientist)
  • Research team

User Requirements

  • Transparent task and job coordination
  • Multi-level access rights and data visibility
  • Secure and efficient collaboration
  • Tracking contributions and project milestones

2. Functional Requirements

  • Secure User Authentication & Role Management: Users log in/register with role-based access (admin, contributor, evaluator).
  • Project Lifecycle Management: Create, assign, and track projects and tasks with status and deadlines.
  • Document Collaboration: Real-time collaborative editing and version control.
  • Contribution Analytics: Visual dashboards showing user contributions, task completion rates, etc.
  • Notifications & Messaging: Alert users to updates, task changes, and deadlines.
  • Multi-granularity Control: Role-specific visibility for fine-grained control of research data.

3. Documents

The platform provides access to a comprehensive set of documents essential for effective citizen science collaboration:

  • Project Proposals: Detailed research project plans, goals, and scopes.
  • Research Papers: Published papers and preprints related to ongoing and past projects.
  • Data Sets: Curated datasets collected from various citizen science contributions.
  • Task Guidelines: Instructions and protocols for task completion at different granularity levels.
  • Meeting Minutes: Summaries of team discussions and decision points.
  • Versioned Documentation: Maintains history of changes to important documents ensuring traceability.

All documents are securely stored and access-controlled to maintain confidentiality and integrity.

4. Implementation

The platform implementation focuses on modular, scalable architecture to accommodate evolving research needs:

  • Technology Stack:
    • Frontend: React with Tailwind CSS for responsive UI
    • Backend: Node.js with Express for API services
    • Database: PostgreSQL for relational data, MongoDB for document storage
    • Authentication: OAuth 2.0 / JWT tokens for secure user access
    • Collaboration: WebSockets for real-time updates and notifications
  • Deployment: Cloud-based (AWS/GCP/Azure) with CI/CD pipelines for seamless updates.
  • Security: End-to-end encryption for data in transit and at rest, multi-factor authentication.
  • Scalability: Microservices architecture enabling modular upgrades and horizontal scaling.
  • Accessibility: Compliance with WCAG 2.1 standards to ensure usability by all researchers.

Future expansions include AI-assisted task assignment and blockchain-based data provenance.

Multi-Granularity Joint Computing Framework

A modern, structured implementation for Citizen Science Project Management

1. Website Purpose & Users

Primary Users

  • Domain Experts: Project leaders guiding research
  • Citizen Scientists: Volunteers collecting and analyzing data
  • Admins: Managing site content and user permissions

User Activities

  • Browsing and searching for citizen science projects
  • Subscribing to and managing project notifications
  • Uploading and sharing scientific documents and reports
  • Participating in collaborative data analysis and discussions

2. Project Requirements

This platform requires a robust and scalable architecture supporting the following:

  • Multi-level user roles and permissions
  • Secure user authentication and session management
  • Advanced search with filtering by project and document attributes
  • Real-time notification system for project updates
  • Collaborative document repository with version control
  • Responsive design for all devices and accessibility compliance

(Note: Integration with third-party scientific data APIs is planned for v2.0)

3. Document Architecture

Entity Types

  • Projects: Core research initiatives
  • Documents: Scientific papers, reports, datasets
  • Users: Registered participants
  • Notifications: Alerts on project activities

Search Capabilities

Search supports:

  • Keyword matching in titles, abstracts, and author names
  • Filters by project domain, date, document type, and status
  • Sorting by relevance, date added, or popularity

Notifications

Notifications will be delivered as:

  • In-app alerts
  • Email summaries (optional)
  • Push notifications on mobile devices (planned)

4. Implementation Plan

Technologies & Frameworks

  • Frontend: React.js with Tailwind CSS
  • Backend: Node.js with Express
  • Database: PostgreSQL for structured data and Elasticsearch for search
  • Authentication: OAuth 2.0 / JWT tokens
  • Hosting: Cloud-based containerized services (AWS / Azure)

Development Phases

  1. Phase 1: Setup project scaffolding, database schema, user auth
  2. Phase 2: Implement project & document upload and management
  3. Phase 3: Develop advanced search & filtering features
  4. Phase 4: Notifications system & user preferences
  5. Phase 5: Mobile responsiveness, accessibility improvements, and testing

5. Project Team Roles

Domain Experts

Lead scientific projects and oversee research quality.

Developers

Build, test, and maintain the software platform.

UI/UX Designers

Ensure intuitive and accessible user interfaces.

Admins

Manage user roles, content moderation, and system monitoring.

A modern, structured implementation for Citizen Science Project Management

Secondary Users

1.2 Platform Purpose

This platform offers a modern multi-granularity structured implementation of a citizen science project management system that supports multiple stakeholders in joint scientific endeavors.

2. User Requirements

  • User roles (Developer, Instructor, Client, Student, Observer) with role-based access and permissions.
  • Comprehensive project documentation and version control for joint research.
  • Real-time collaboration and commenting on research items.
  • Multiple granularity levels for research artifacts management.
  • Integration with existing academic resources and citation management.
  • Extensible plugin system for future features and extensions.
  • Responsive design for mobile and desktop access.

3. Documents Provided

  • Requirements specification document
  • ER Model document
  • Platform Documentation Document

4. Implementation Overview

The platform was implemented with a focus on modularity, scalability, and usability. Core technologies include React for frontend UI, Node.js and Express for backend API, and PostgreSQL for the database. The system uses OAuth 2.0 for authentication and supports role-based authorization.

Key Features

  • Multi-level project granularity management
  • Version-controlled collaborative editing
  • Real-time notifications and comments
  • Responsive UI with accessibility standards
  • Secure role-based authentication and authorization
  • Integration with academic citation services

Multi-Granularity Joint Computing Framework

A modern, structured implementation for Citizen Science Project Management

1. Website Purpose & Users

Primary Users

  • Development team (5 MIS students)
  • Instructor (for evaluation)
  • Client (scientist)
  • Research team

User Requirements

  • Transparent task and job coordination
  • Multi-level access rights and data visibility
  • Secure and efficient collaboration on research documentation
  • Tracking contributions and project milestones

2. Functional Requirements

  • User registration and authentication with role management
  • Project and task creation, assignment, and tracking
  • Hierarchical data management for multi-granularity documentation
  • Real-time notifications and status updates
  • Version control for project documents
  • Reporting and analytics dashboard

3. Documents & Tasks Management

The platform supports various document types including:

  • Research Proposals
  • Field Data Collection Sheets
  • Experiment Results and Reports
  • Collaborative Notes and Annotations
  • Publication Drafts

Task management is integrated with documents, enabling multi-level tracking from individual data entries up to aggregated project summaries.

4. Implementation Notes

The platform is implemented as a web application leveraging:

  • Modern frontend frameworks with responsive design
  • RESTful APIs for backend communication
  • Robust database schemas supporting multi-granularity data
  • Secure authentication and authorization mechanisms
  • Real-time collaboration features using WebSocket or similar tech

Additional considerations include performance optimization for large datasets and modular design for future extensibility.
